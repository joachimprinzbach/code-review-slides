Code Reviews

Was ist ein Code Review?

Qualität / Clean Code

  • Konsistenz: Namen, Style, Abstraktionen
  • Gutes Design, ausreichend Tests
  • Fehler möglichst früh finden: Auch Security oder Threading Probleme
  • Anforderungen sind korrekt umgesetzt
  • Psychologischer Effekt des Reviews: Wenn man weiss, dass es jemand anschaut...

Austausch / Wissen

  • Fördert Kommunikation und Austausch
  • Vier Augen Prinzip
  • Mindestens 2 Entwickler kennen jede Code Stelle
  • Lesen von Code dient der Weiterbildung

How to Code Review

  • gemeinsam an einem PC / über Video
  • Code auschecken und selbst lesen
  • mit Hilfe von Tools -> PullRequest

Automation

  • Static Code Analysis (Sonarqube, ESLint)
  • Security Checks (Nexus IQ Server, XRay, Checkmarx, Clair)
  • Code Formatting (your IDE, prettier)
  • Testing (JUnit, Cypress.io, Selenium, Karma, Jest)
  • Architecture Checks (ArchUnit, JDepend)
  • Continuous Integration (Jenkins, GitLab CI/CD, Travis CI, Circle CI, GitHub Actions)

Best Practices

  • Immer wertschätzend bleiben.
  • Automatisieren wo möglich.
  • mit Review von Tests starten -> diese zeigen die Verwendung.
  • Reviews "erzwingen" -> keine Änderungen ohne abgenommenen PullRequest.
  • Ihr dürft auch Lob verteilen.
  • Pairing ist ebenfalls ein tolles Code Review Instrument.
